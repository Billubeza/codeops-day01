\# Reflection Answers



\### 1. What is the difference between Git and GitHub?

\* \*\*Git\*\* is a local version control system tool that tracks changes in your code on your machine. 

\* \*\*GitHub\*\* is a cloud-based hosting service that stores your Git repositories online, allowing for collaboration, backup, and sharing.



\### 2. What is the purpose of the terminal?

The terminal is a text-based interface used to interact directly with your computer's operating system. It allows you to execute commands, manipulate files, and automate tasks faster than a graphical user interface (GUI).



\### 3. What does git add do?

`git add` moves changes from your working directory to the "Staging Area". It tells Git which modified or new files you want to include in your next snapshot (commit).



\### 4. What is the purpose of git commit?

`git commit` takes a permanent snapshot of your staged changes and saves it to the local repository history. It acts like a save point, accompanied by a message describing what changed.



\### 5. Write the daily Git workflow in the correct order.

1\. `git pull` (to fetch and merge latest remote changes)

2\. Make code modifications / create files

3\. `git status` (to review changes)

4\. `git add <file\_names>` (to stage changes)

5\. `git commit -m "your message"` (to save locally)

6\. `git push origin <branch\_name>` (to upload online)
